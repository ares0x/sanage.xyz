# 潜伏版人格测试：一天，12k 访问量

上周四下午，我在深圳太子湾 K11 参加一个开发者组织的活动。中途，我女朋友从小红书发来一个帖子，是一位博主 @自动挡赛车手 用 Kimi Code 做的《大明王朝1566》人物人格测试。

我点进去玩了一下感觉很有意思，很容易让人产生分享欲。我很快意识到：  当人格测试绑定具体角色时，它天然就是一种“可分享的内容”。 于是我随口说了一句：那我今天做个《潜伏》版的。

这句话，后来变成了一次完整的产品实验。

## 一、我在做一次“快速验证”
整个开发过程，我几乎完全依赖 Kimi 和 Kimi Code。Kimi 帮我做三件事：

- 聊需求，确定技术栈
- 拆解《潜伏》中的核心人物
- 构建人格维度和角色映射逻辑

这套技术栈的核心优势是：几乎没有开发成本，且可以在极短时间内上线验证。
在整个项目开发过程中，我只是花 15元买了 sanage.xyz 这个域名。（kimi code 是在开发这个项目之前购买的，不计入其中）
Kimi Code 则负责在我确定技术栈之后，直接参与代码生成和页面搭建。项目最终部署在 Cloudflare Pages 上。
我刻意做了几个比较“克制”的选择：

- 不做复杂后端
- 不做用户系统
- 使用 Cloudflare 托管

只保留一件事：用户愿不愿意玩，以及会不会分享。 本质上这是在用最小成本验证一个假设：用户会不会因为测试结果而产生分享行为。

从动手到初版上线，用时很短。周五，我把它发在了小红书和抖音。

## 二、数据没有增长，直到第4天

上线后的前三天，几乎没有反馈，只有身边的熟人测试过。在小红书和抖音各发了一篇图文内容，同样的内容，小红书只有 41 次浏览，而抖音在第 4 天开始放量。在第 4 天抖音的图文开始陆陆续续出现了一些观看，评论区开始有人评论要测试链接。Cloudflare 的 Web Analytics显示，从昨天上午9点，到今天上午9点，这个测试带来了 12.1k 的访问量。评论区现在仍然不断有人要链接，也出现了用户自发分享测试结果的情况。

同时，我也第一次非常真切地感受到平台机制的“另一面”：
- 评论区频繁发链接 → 被限制为仅好友可见
- 私信回复链接 → 很快触发频率上限

当流量开始增长时，分发能力反而受限于平台规则。后续我又在抖音陆续发了两篇图文，一篇是介绍测试版本更新，另外一篇是介绍创作思路。这两篇到现在也有 11k 和 4k 的观看量。一个有意思的点是在这两篇的文尾都有测试链接，即使我在内容里已经放了链接，用户仍然更倾向于在评论区“索要链接”。 **用户行为并不总是理性的，而是更依赖平台习惯。**

## 三、思考和观察
几个有意思的观察：
- AI 不只是降低开发成本，某种意义上来说，它缩短了“从想法到验证”的路径。
- 抖音依然是情绪的放大器：抖音用户对 “测试/人格” 类互动的参与度极高。（抖音没有小红书那么严苛的外链管理，但过于频繁地发链接依然会被吞评论以及限制评论功能）
- 大家永远乐于探索未知的自己，哪怕这只是个娱乐向的测试。“人格测试” 之所以容易传播，是因为它提供了一个低成本的自我表达标签。
- 在自媒体平台宣传作品一定要在平台规则范围内，提前设计好产品入口的位置。

这次《潜伏》测试，不算是一个非常成功的产品尝试反倒像一次不完整的增长实验。测试入口： [QuizSanage](quiz.sanage.xyz) 欢迎体验。
希望未来可以做出更多好玩好用的产品出来，共勉。
